# Multi-Tool Research Agent

一个**多工具研究型 AI Agent**：面对需要"查资料 + 计算 + 记录"的复杂问题，它能**自主判断该用哪个工具、按什么顺序使用**，一步步推进到最终答案。

这是在单工具 Agent 基础上的进阶项目，核心是 **多工具编排（Multi-Tool Orchestration）与自主工具选择**——Agent 拥有 3 个工具，每一步由模型自己决定调用哪一个。采用 **ReAct 模式（Reasoning + Acting）**：在"思考 → 行动 → 观察"的循环中推进任务。

## Agent 的三个工具

| 工具 | 作用 | 实现 |
|------|------|------|
| `web_search` | 联网搜索信息 | DuckDuckGo + 维基百科（双重备用，无需 API key）|
| `calculator` | 安全数学计算 | AST 解析求值（不使用危险的 eval）|
| `save_note` | 保存结论到文件 | 写入本地 txt |

## 效果演示

```
$ python agent.py "特斯拉和比亚迪2024年营收差多少？"

问题：特斯拉和比亚迪2024年营收差多少？
=======================================================

[第1步·思考] 我需要先查这两家公司2024年的营收。
[调用工具] web_search({'query': 'Tesla 2024 revenue'})
[工具结果] ...

[第2步·思考] 再查比亚迪的营收。
[调用工具] web_search({'query': 'BYD 2024 revenue'})
[工具结果] ...

[第3步·思考] 现在计算两者差值。
[调用工具] calculator({'expression': '97600 - 107200'})
[工具结果] 97600 - 107200 = -9600

[第4步·思考] 得出结论，保存。
[调用工具] save_note({'content': '...'})

[最终回答]
根据查询，两家公司2024年营收差约为 ...
=======================================================
```

## 工作原理（ReAct 循环）

```
用户问题
   ↓
┌─────────────────────────────────┐
│  模型思考：现在该用哪个工具？      │ ←──┐
│      ↓                          │    │
│  调用工具（搜索/计算/保存）        │    │ 循环，直到
│      ↓                          │    │ 得出最终答案
│  观察工具返回的结果               │ ───┘
└─────────────────────────────────┘
   ↓
给出最终回答
```

与单工具 Agent 的关键区别：模型不再只是"调用唯一的工具"，而是要在**多个工具之间做选择和编排**——这正是智能体自主决策能力的核心。

## 技术栈

- Python
- OpenRouter API（兼容 OpenAI 接口，使用免费模型）
- Function Calling / Tool Use
- ReAct 模式（多步推理 + 行动）
- AST 安全求值

## 使用方法

```bash
pip install -r requirements.txt
export OPENROUTER_API_KEY='你的key'
python agent.py "你的问题"
```

## 设计亮点

- **多工具自主选择**：模型每一步自行决定用哪个工具，而非写死流程
- **安全计算**：用 AST 解析代替 `eval`，避免代码注入风险
- **搜索容错**：DuckDuckGo 失败时自动回退到维基百科
- **可扩展**：新增工具只需在 `TOOL_FUNCS` 和 `TOOLS` 注册即可

## 后续可扩展方向

- 增加更多工具（汇率换算、日期计算、读取本地文件）
- 接入更强的搜索 API 提升信息质量
- 加入对话记忆，支持连续追问
